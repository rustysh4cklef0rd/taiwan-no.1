# Design: Traditional Chinese Reading Widget for Android

## Problem Statement

A Mandarin speaker (Traditional Chinese) who can already speak and understand the language
wants to learn to read and write. Most language apps assume you're a total beginner — teaching
tones, pronunciation, meaning from scratch. This person needs none of that. They need character
recognition only.

The solution: an Android home screen widget that shows 6 curated characters per day, drawn
from a ~1,000 word core frequency list. No willpower required. No app to open. It's just there.

## What Makes This Cool

The ambient learning mechanic. Every time you glance at your phone — which you do 80+ times
a day — you see 6 characters. No habit to build. No streak to maintain. The widget IS the
learning loop. 6 words × ~170 days = reading fluency in under 6 months.

EUREKA (logged): Every language app assumes users need gamification (streaks, points). This
user is a fluent Mandarin speaker — they have intrinsic motivation. No gamification needed.
Simpler product, stronger positioning. Differentiation through subtraction.

This is genuinely underserved: apps like HelloChinese and Duolingo treat Mandarin speakers
as beginners. Pleco is a dictionary, not a curriculum. This fills the gap.

## Constraints

- Android-only for v1 (Flutter architecture keeps iOS as a future option — TODOS.md)
- User already knows spoken Mandarin — audio/pinyin reinforce character-to-sound, not teach meaning
- Word list is static/bundled JSON for v1. No server, no sync, no personalization.
- Widget is the core product; companion app handles setup/config only
- No gamification (no streaks, no points) — ambient exposure is the mechanic

## Premises (all confirmed)

## Recommended Approach: Flutter + home_widget

### Tech Stack
- `home_widget` (pub.dev) — renders Android AppWidget from Flutter
- `flutter_tts` — Text-to-Speech for pronunciation audio
- `shared_preferences` — persist word list state and widget data
- Word list: JSON file bundled in assets/, ~1,000 entries frequency-ordered (SUBTLEX-CH source)
- Daily word selection: deterministic (epochDay % totalWords, gives 6 sequential words)
- Companion app: minimal — first-run onboarding, settings (theme, font size)

### Word Entry Schema (complete)
```json
{
  "id": 42,
  "character": "好",
  "pinyin": "hǎo",
  "meaning": "good / well",
  "phrase": "你好",
  "phrase_pinyin": "nǐ hǎo",
  "phrase_meaning": "hello / how are you",
  "frequency_rank": 15
}
```

### Daily Word Rotation
- Selection: `startIndex = (epochDay * 6) % totalWords` → returns words[startIndex..startIndex+5]
- Deterministic: same date always shows same words across devices/reinstalls
- Wrap behavior: after completing all ~1,000 words (~167 days), index wraps to beginning
  (review cycle — desirable for reinforcement)
- Year boundary: no special case needed; epochDay is continuous across year boundaries

### Background Update Architecture
```
WorkManager (daily trigger, midnight)
    │
    ├── Dart background isolate (via home_widget callback)
    │   ├── ✓ Read today's date
    │   ├── ✓ Compute word indices (deterministic)
    │   ├── ✓ Write word data to SharedPreferences
    │   ├── ✓ Call HomeWidget.saveWidgetData()
    │   └── ✗ NO audio playback (method channel requires UI thread)
    │       ✗ NO UI operations (isolate limitation)
    │
    └── Widget re-renders with new data from SharedPreferences
```

**CRITICAL:** Background Dart isolates cannot use method channels that require the UI
thread. Audio MUST only play from foreground (user tap). Any attempt to play TTS in
the background will crash silently or hang.

### Tap Flow
```
User taps widget character
    │
    ├── Widget fires PendingIntent → launches companion app
    │   └── Detail view opens with word data passed as Intent extras
    │       ├── Shows: character (large), pinyin, meaning, example phrase
    │       ├── Speaker button → flutter_tts.speak(character + phrase)
    │       └── TTS check: if unavailable → speaker button disabled, show "Install Chinese TTS"
    │
    └── User returns to home screen
```

### TTS Availability
- On first launch: check `flutter_tts.isLanguageAvailable("zh-TW")` (Traditional Chinese)
- If unavailable: show one-time prompt "For audio pronunciation, install Chinese TTS from Settings"
- Graceful fallback: all text (character, pinyin, phrase) still visible; speaker icon grayed out
- Never crash on TTS unavailability

### First-Run Initialization
1. App launches → check if word list is initialized in SharedPreferences
2. If not: copy words.json from assets/ → parse → store word list metadata
3. Schedule WorkManager daily task
4. Trigger initial widget update (don't wait until midnight for first display)

## Design Spec

### Widget Information Hierarchy
```
┌─────────────────────────────────────────┐
│  好      nǐ hǎo (phrase)                │  ← word 1 (char dominant)
│  大      dà                              │  ← word 2
│  小      xiǎo                            │  ← word 3
│  人      rén                             │  ← word 4
│  来      lái                             │  ← word 5
│  去      qù                              │  ← word 6
└─────────────────────────────────────────┘
Layout: 4×2 grid (primary) + 2×2 grid (3 words, secondary)
```

**Typography:**
- Character: Noto Serif CJK TC, Bold, 28sp (4×2) / 36sp (2×2)
- Pinyin: Noto Sans, Regular, 12sp / 14sp
- Background: #FFFFFF light / #1A1A1A dark
- Character color: #1A1A1A light / #F5F5F5 dark
- Pinyin color: #666666 light / #AAAAAA dark
- Aesthetic: minimal/clean — let the characters be the art

**Widget Sizes:**
- 4×2 (default): shows 6 words, character + pinyin
- 2×2 (compact): shows 3 words, character only (pinyin on tap)
- Minimum touch target: 48dp per character cell (Android accessibility standard)

**Tap Detail View:**
```
┌─────────────────────────┐
│                         │
│         好              │  ← character, Noto Serif CJK TC, 96sp
│         hǎo             │  ← pinyin, 24sp
│      good / well        │  ← meaning, 16sp
│                         │
│   ┌──────────────────┐  │
│   │  你好 • nǐ hǎo   │  │  ← example phrase card
│   │  hello / hi      │  │
│   └──────────────────┘  │
│            🔊           │  ← speaker button (48dp touch target)
└─────────────────────────┘
```

**Empty/Loading States:**
- Widget before first update: "Loading today's words..." with a subtle shimmer
- First-launch onboarding: 2-screen companion app (language selection confirm, widget setup instructions)
- Zero TTS: speaker icon shown but grayed out with tooltip "Install Chinese TTS"

**Dark mode:**
- Widget background: #1A1A1A
- Character: #F5F5F5
- Pinyin: #AAAAAA
- Detail view follows system dark mode automatically

## Implementation Steps (complete, ordered)

1. **Source word list**: Download SUBTLEX-CH frequency corpus → filter to 1,000 Traditional
   Chinese entries → generate words.json with complete schema (character, pinyin, meaning,
   phrase, phrase_pinyin, phrase_meaning, frequency_rank)

2. **Scaffold project**: `flutter create chinese_reading_widget --org com.chinesewidget`

3. **Add dependencies to pubspec.yaml**:
   - `home_widget: ^0.4.0`
   - `flutter_tts: ^4.0.0`
   - `shared_preferences: ^2.2.0`
   - `workmanager: ^0.5.0`
   - Font: `google_fonts: ^6.0.0` (Noto Serif CJK TC)

4. **Build data layer**:
   - `lib/models/word.dart` — Word class with fromJson/toJson
   - `lib/services/word_service.dart` — loadWordList(), getTodaysWords(DateTime), checkTtsAvailability()
   - `assets/data/words.json` — bundled word list

5. **Build widget layout (Android)**:
   - `android/.../widget_layout_4x2.xml` — 6-word layout
   - `android/.../widget_layout_2x2.xml` — 3-word layout
   - `android/.../WordWidgetProvider.kt` — AppWidgetProvider with WorkManager scheduling
   - Typography: Noto Serif CJK TC (include as asset font)

6. **Build Flutter foreground layer**:
   - `lib/main.dart` — app entry point, first-run initialization
   - `lib/screens/detail_screen.dart` — tap detail view (character, pinyin, phrase, TTS)
   - `lib/screens/onboarding_screen.dart` — 2-screen setup flow
   - `lib/widget/background_callback.dart` — WorkManager Dart callback (data-write only)

7. **Wire up daily rotation**:
   - Register WorkManager task on first launch (daily, ~midnight)
   - Implement deterministic word selection (epochDay-based)
   - Trigger immediate widget update on first launch (don't wait until midnight)

8. **Implement TTS**:
   - `flutter_tts.setLanguage("zh-TW")` for Traditional Chinese
   - Availability check on launch → graceful fallback if unavailable
   - Play character pronunciation + example phrase on detail view

9. **Add progress indicator**:
   - Companion app settings screen: "Day 34 of 167 — you're 20% through"
   - Optional: small progress arc on widget (non-intrusive)

10. **Dark mode support**:
    - Widget layout uses night-mode XML variants
    - Detail view uses Flutter's `ThemeData` with system brightness detection

11. **Write tests** (see test plan below — 18 paths, target 100% coverage)
